
English
--------------------------------------------------------------------------------

If OOHG package was installed in C:\OOHG, and MinGW is going to be used to
compile projects, there's no need to change the default configuration. Simply
add C:\OOHG to the system's PATH and use DOS commands "COMPILE prg_name" or
"COMPILE_MINGW prg_name" to build executable files. If you don´t want to change
the PATH, use "C:\OOHG\COMPILE prg_name" or "C:\OOHG\COMPILE_MINGW prg_name"
instead.

If OOHG package was installed in another folder (e.g. D:\XYZ), you must set
the following environment variable, before using the compilation commands:

set HG_ROOT=D:\XYZ

By default OOHG package uses the folder "LIB" to store it's libraries.
To use a different folder (e.g. C:\OOHG\LIB\WIN\MINGW), set the following
environment variable, before using the compilation commands:

set LIB_GUI=C:\OOHG\LIB\WIN\MINGW

By default OOHG package uses a bundled version of Harbour to compile projects.
To use a different version of Harbour (e.g. installed in D:\MYHRB), set the
following environment variable, before using the compilation commands:

set HG_HRB=D:\MYHRB

If the new version stores it's libraries in a folder different than
"D:\MYHRB\LIB" (e.g. at D:\MYHRB\LIB\WIN\MINGW), set the following environment
variable, before using the compilation commands:

set LIB_HRB=D:\MYHRB\LIB\WIN\MINGW

If the new version stores it's binaries in a folder different than
"D:\MYHRB\BIN" (e.g. at D:\MYHRB\BIN\WIN\MINGW), set the following environment
variable, before using the compilation commands:

set BIN_HRB=D:\MYHRB\BIN\WIN\MINGW

By default OOHG package uses a bundled version of MinGW to compile projects.
To use a different version of MinGW (e.g. installed in D:\MYMINGW), set the
following environment variable, before using the compilation commands:

set HG_MINGW=D:\MYMINGW

To use Borland C++ 5.5 compiler instead of MinGW, edit C:\OOHG\COMPILE.BAT,
locate the line "call %HG_ROOT%\compile_mingw.bat %1 %2 %3 %4 %5 %6 %7 %8 %9"
and replace "compile_mingw" with "compile_bcc". If the compiler is not installed
at C:\BORLAND\BCC55 (e.g. at D:\MYBCC), set the following environment variable,
before using the compilation commands:

set HG_BCC=D:\MYBCC

To use Pelles C compiler instead of MinGW, edit C:\OOHG\COMPILE.BAT,
locate the line "call %HG_ROOT%\compile_mingw.bat %1 %2 %3 %4 %5 %6 %7 %8 %9"
and replace "compile_mingw" with "compile_pc". If the compiler is not installed
at C:\PELLESC (e.g. at D:\PCC), set the following environment variable,
before using the compilation commands:

set HG_PC=D:\PCC

To use Microsoft Visual Studio 9.0 compiler instead of MinGW, edit
C:\OOHG\COMPILE.BAT, locate the line
"call %HG_ROOT%\compile_mingw.bat %1 %2 %3 %4 %5 %6 %7 %8 %9"
and replace "compile_mingw" with "compile_vc". If the compiler is not installed
at %PROGRAMFILES%\MICROSOFT VISUAL STUDIO 9.0\vc (e.g. at D:\VS9), set the
following environment variable, before using the compilation commands:

set HG_VC=D:\VS9


Español
--------------------------------------------------------------------------------

Si el paquete OOHG se instala en C:\OOHG y MinGW va a ser utilizado para
compilar los proyectos, no hay necesidad de cambiar la configuración por
defecto. Simplemente añada C:\OOHG al PATH del sistema y utilice los comandos
de DOS "Compilar prg_name" o "COMPILE_MINGW prg_name" para crear los archivos
ejecutables. Si no desea cambiar el PATH, utilice "C:\OOHG\COMPILE prg_name" o
"C:\OOHG\COMPILE_MINGW prg_name" en su lugar.

Si el paquete OOHG se instaló en otra carpeta (por ejemplo D:\XYZ), deberá
setear la siguiente variable de ambiente, antes de usar los comandos de
compilación:

set HG_ROOT=D:\XYZ

Por defecto, OOHG utiliza la carpeta "LIB" para guardar sus librerías.
Para utilizar otra carpeta (por ej. C:\OOHG\LIB\WIN\MINGW), deberá setear la
siguiente variable de ambiente, antes de usar los comandos de compilación:

set LIB_GUI=C:\OOHG\LIB\WIN\MINGW

Por defecto el paquete OOHG utiliza la versión incluida de Harbour para
compilar los proyectos. Para utilizar una versión diferente (por ej. la
instalada en D:\MYHRB), deberá setear la siguiente variable de ambiente,
antes de usar los comandos de compilación:

set HG_HRB=D:\MYHRB

Si la nueva versión guarda sus librerías en una carpeta distinta a
"D:\MYHRB\LIB" (por ej. en D:\MYHRB\LIB\WIN\MINGW), deberá setear la siguiente
variable de ambiente, antes de usar los comandos de compilación:

set LIB_HRB=D:\MYHRB\LIB\WIN\MINGW

Si la nueva versión guarda sus binarios en una carpeta distinta a
"D:\MYHRB\BIN" (por ej. en D:\MYHRB\BIN\WIN\MINGW), deberá setear la siguiente
variable de ambiente, antes de usar los comandos de compilación:

set BIN_HRB=D:\MYHRB\BIN\WIN\MINGW

Por defecto, el paquete OOHG utiliza la versión incluida del compilador MinGW.
Para utilizar una versión diferente de MinGW (por ejemplo la instalada en
D:\MYMINGW), deberá setear la siguiente variable de ambiente antes de usar los
comandos de compilación:

set HG_MINGW=D:\MYMINGW

Para utilizar el compilador Borland C++ 5.5 en lugar de MinGW, edite
C:\OOHG\COMPILE.BAT, localice la línea
"call %HG_ROOT%\compile_mingw.bat %1 %2 %3 %4 %5 %6 %7 %8 %9" y reemplace
"compile_mingw" con "compile_bcc". Si el compilador no está instalado en 
C:\BORLAND\BCC55 (por ejemplo, está en D:\MYBCC), deberá setear la siguiente
variable de ambiente antes de usar los comandos de compilación:

set HG_BCC=D:\MYBCC

Para utilizar el compilador Pelles C en lugar de MinGW, edite
C:\OOHG\COMPILE.BAT, localice la línea
"call %HG_ROOT%\compile_mingw.bat %1 %2 %3 %4 %5 %6 %7 %8 %9" y reemplace
"compile_mingw" con "compile_pc". Si el compilador no está instalado en
C:\PELLESC (por ejemplo, está en D:\PCC), deberá setear la siguiente
variable de ambiente antes de usar los comandos de compilación:

set HG_PC=D:\PCC

Para utilizar el compilador Microsoft Visual Studio 9.0 en lugar de MinGW, edite
C:\OOHG\COMPILE.BAT, localicela línea
"call %HG_ROOT%\compile_mingw.bat %1 %2 %3 %4 %5 %6 %7 %8 %9" y reemplace
"compile_mingw" con "compile_vc". Si el compilador no está instalado en
%PROGRAM FILES%\MICROSOFT VISUAL STUDIO 9.0\vc (por ejemplo, está en D:\VS9),
deberá setear la siguiente variable de ambiente antes de usar los comandos de
compilación:

set HG_VC=D:\VS9
